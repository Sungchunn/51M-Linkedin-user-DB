INSIGHT API Documentation

Hybrid talent search API optimized for HTTP request nodes. Supports GET with query params and POST with JSON.

Base URL

http://localhost:8000

Authentication

Use an API key for server-to-server usage. Send header X-API-Key: <your-key>. Tiers and scopes:

  • Public (no key): limit ≤ 50, no exports, PII redacted.
  • Basic key: limit ≤ 200, PII redacted, no exports unless scope export:read.
  • Trusted key: limit ≤ 1000, exports allowed with export:read, optional pii:read for PII fields.

Rate Limits

  • Search: 60/min (burst 120)
  • Export: 6/min (burst 10)

Using HTTP Request Nodes

Configure the node as follows:

  • Method: GET (for simple searches) or POST (for complex JSON)
  • URL: http://localhost:8000/search
  • Params: Add repeated keys for arrays (e.g., skills)
  • Timeout: 30s recommended

Example cURL (mirrors most HTTP request nodes):

curl -G \
  --data-urlencode "q=senior python engineer" \
  --data-urlencode "location_country=united states" \
  --data-urlencode "regions=california" \
  --data-urlencode "skills=python" \
  --data-urlencode "skills=sql" \
  "http://localhost:8000/search"

Performance & Limits

  • Pagination: limit max 1000, use offset for paging.
  • Query timeout: 30s server-side; keep requests focused.
  • Caching: Filters endpoints cache for 10 minutes via Cache-Control.

Bulk Export

GET /export/ndjson

Return up to 1000 results per call in NDJSON format (one JSON object per line). Requires export:read. Non-empty q and at least one filter (e.g., location_country) are required.

GET /export/ndjson?q=python&limit=1000®ions=california
200 OK
{...}\n
{...}\n
{...}\n

Recommended for workflows where you need 1000 leads per call. Use offset to paginate subsequent exports.

GET /export/csv

Return up to 1000 results as a CSV file (streamed). Requires export:read. Non-empty q and at least one filter are required. PII fields are redacted unless the key has pii:read.

GET /export/csv?q=python&limit=1000®ions=california
200 OK (Content-Type: text/csv)
id,full_name,job_title,company_name,industry,location_country,region,locality,skills,score,...
...

Next Page Tokens

Responses from /search include next_page_token when more results exist. Pass it back using page_token to fetch the next page without managing offsets.

GET /search?q=python&limit=1000
200 OK
{
  "results": [...],
  "total_count": 12345,
  "returned_count": 1000,
  "next_page_token": "eyJxdWVyeSI6ICJweXRob24iLCAi..." 
}

# Next page
GET /search?page_token=eyJxdWVyeSI6ICJweXRob24iLCAi...
200 OK
{ "results": [...], ... }
