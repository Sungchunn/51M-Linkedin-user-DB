Introduction

The PROSPECTIQ API provides programmatic access to semantic talent search across 497K+ LinkedIn profiles. Our hybrid search combines vector embeddings (80%) with traditional lexical matching (20%) for optimal relevance.

Base URL
http://localhost:8000

Key Features

  • Semantic Search: Natural language queries using OpenAI embeddings
  • Advanced Filters: Location, skills, experience, industry, and more
  • Bulk Export: CSV and NDJSON formats for CRM integration
  • Real-time Performance: Sub-second query responses
  • Flexible Authentication: API keys with custom scopes and rate limits

Authentication

API keys are required for production usage. Include your API key in the X-API-Key header.

API Key Tiers

Tier Max Results Rate Limit Features
Public 50 60/min Basic search, PII redacted
Basic 200 200/min Search + Export (with scope)
Trusted 1000 1000/min Full access + PII (with scope)

Scopes

  • search:read - Basic search access (default)
  • export:read - CSV/NDJSON export capabilities
  • pii:read - Access to email, phone, LinkedIn URLs
Example Request
curl http://localhost:8000/search?q=engineer \
  -H "X-API-Key: your-api-key-here"

Rate Limits

Rate limits are enforced per IP address (for public) or per API key (for authenticated requests).

Endpoint Limit Burst
/search 60/min (public)
200/min (basic)
1000/min (trusted)		 120/min
/export/* 6/min 10/min
/filters No limit -
⚠️ Rate Limit Headers
When rate limited, you'll receive a 429 Too Many Requests response. Retry after the specified time in the Retry-After header.

Quick Start

Get started with the PROSPECTIQ API in 3 simple steps.

Step 1: Get an API Key

Search Profiles

Semantic search with advanced filtering. Supports both GET (query params) and POST (JSON body).

GET /search

Search using URL query parameters. Ideal for simple integrations and HTTP request nodes.

Query Parameters

Parameter Type Description Default
q string Search query (natural language) ""
limit integer Results per page (1-1000) 20
offset integer Pagination offset 0
location_country string Filter by country -
regions array Filter by regions (repeat param) -
localities array Filter by cities (repeat param) -
industries array Filter by industries (OR logic) -
skills array Required skills (AND logic) -
min_years_experience integer Minimum years (0-80) -
max_years_experience integer Maximum years (0-80) -
job_title string Filter by job title (partial match, case-insensitive) -
company string Filter by company name (partial match, case-insensitive) -
has_linkedin boolean Filter profiles with LinkedIn URL -
has_email boolean Filter profiles with email address -
has_phone boolean Filter profiles with phone number -
has_website boolean Filter profiles with website/domain -
has_twitter boolean Filter profiles with Twitter handle -
has_github boolean Filter profiles with GitHub username -

Example Request

GET /search?q=senior+python+engineer&location_country=united+states®ions=california&skills=python&skills=sql&job_title=software+engineer&has_email=true&limit=20

Export Data

Bulk export search results in CSV or NDJSON format. Requires export:read scope.

⚠️ Export Requirements
Export endpoints require:
  • API key with export:read scope
  • Non-empty query (q) OR at least one filter
  • Maximum 1000 results per request
GET /export/csv

Export search results as CSV. PII fields redacted unless key has pii:read scope.

Example Request

GET /export/csv?q=python&limit=1000®ions=california
Content-Type: text/csv; charset=utf-8
Content-Disposition: attachment; filename="export.csv"

id,full_name,job_title,company_name,industry,location,skills,score
uuid,John Doe,Senior Engineer,Tech Corp,Software,"San Francisco, CA","[Python, SQL]",0.92
...
GET /export/ndjson

Export search results as NDJSON (newline-delimited JSON). One JSON object per line.

Example Response

{"id":"uuid","full_name":"John Doe","job_title":"Senior Engineer",...}
{"id":"uuid","full_name":"Jane Smith","job_title":"Staff Engineer",...}
{"id":"uuid","full_name":"Bob Wilson","job_title":"Principal Engineer",...}

Pagination

Use offset-based pagination or opaque page tokens for efficient result navigation.

Offset-Based Pagination

// Page 1
GET /search?q=engineer&limit=20&offset=0

// Page 2
GET /search?q=engineer&limit=20&offset=20

// Page 3
GET /search?q=engineer&limit=20&offset=40

Token-Based Pagination

Use next_page_token for easier pagination without managing offsets. 

// First request
GET /search?q=python&limit=100

{
  "results": [...],
  "next_page_token": "eyJxdWVyeSI6..."
}

// Next page
GET /search?page_token=eyJxdWVyeSI6...

Code Examples

cURL

curl -G "http://localhost:8000/search" \
  -H "X-API-Key: your-api-key" \
  --data-urlencode "q=senior software engineer" \
  --data-urlencode "location_country=united states" \
  --data-urlencode "regions=california" \
  --data-urlencode "skills=python" \
  --data-urlencode "job_title=software engineer" \
  --data-urlencode "has_email=true" \
  --data-urlencode "limit=50"

JavaScript (fetch)

const response = await fetch('http://localhost:8000/search', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'X-API-Key': 'your-api-key'
  },
  body: JSON.stringify({
    query: 'senior software engineer',
    location_country: 'united states',
    regions: ['california'],
    skills: ['python', 'react'],
    job_title: 'software engineer',
    has_email: true,
    has_linkedin: true,
    limit: 50
  })
});

const data = await response.json();
console.log(`Found ${data.total_count} results`);
console.log(data.results);

Python (requests)

import requests

response = requests.post('http://localhost:8000/search',
    headers={'X-API-Key': 'your-api-key'},
    json={
        'query': 'senior software engineer',
        'location_country': 'united states',
        'regions': ['california'],
        'skills': ['python', 'react'],
        'job_title': 'software engineer',
        'has_email': True,
        'has_linkedin': True,
        'limit': 50
    }
)

data = response.json()
print(f"Found {data['total_count']} results")
for profile in data['results']:
    print(f"{profile['full_name']} - {profile['job_title']}")

Error Handling

The API uses standard HTTP status codes. All error responses include a JSON body with details.

Status Code Meaning Common Causes
400 Bad Request Invalid parameters, malformed JSON
401 Unauthorized Missing or invalid API key
403 Forbidden Insufficient scopes or tier
422 Unprocessable Entity Validation errors, offset too high
429 Too Many Requests Rate limit exceeded
500 Internal Server Error Server-side error

Error Response Format

{
  "error": "Invalid query parameter",
  "detail": "limit must be between 1 and 1000",
  "timestamp": "2025-10-21T03:00:00Z"
}
